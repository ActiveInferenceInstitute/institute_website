# Releasing & Versioning

How and where this site is versioned, and the exact steps to cut a release. See
also [INDEX.md](INDEX.md) (How to version & improve) and
[CHANGELOG.md](CHANGELOG.md).

## Where the version lives and flows

`package.json` `version` is the **single source of truth**. From there:

- `src/data.mjs` reads it as `SITE_VERSION` (`src/data.mjs:22`).
- `src/build.mjs` writes `version.json` (`src/build.mjs:103-113`) containing
  `{site_version, built_at, exported_at, source_fingerprint, pages, commit}`,
  where `source_fingerprint` comes from `data/export-manifest.json`, and stamps
  the version into every rendered page.
- `feed.json` / `feed.xml` are generated by `src/feeds.mjs` and carry no version
  string.

The gate `check_site_contract.py` (`check:site`) asserts
`version.json.site_version == package.json.version`, so the version **must** be
bumped and the site **rebuilt** in the same change — bumping one without the
other fails `npm run check`.

## SemVer policy for this content site

- **patch (x.y.Z):** copy edits, link/data fixes, dependency-free tweaks —
  content changed, rebuilt.
- **minor (x.Y.0):** new pages, sections, registries, or a new locale.
- **major (X.0.0):** URL-taxonomy change, domain/baseUrl cutover, or anything
  requiring redirects (precedent: `v2.0.0` was a hard IA break).

## Release checklist

1. Make content/source changes under `src/` (never edit built `*.html`).
2. `node src/build.mjs` to regenerate output.
3. `npm run check` — all gates green.
4. Commit source **and all regenerated HTML together** in one commit (CI
   enforces committed HTML == `build(source)`; the build stamp changes every page
   on a version bump, so a partial commit fails CI).
5. Bump `package.json` `version`; rebuild so `version.json` refreshes.
6. Move `## Unreleased` CHANGELOG entries under a dated
   `## vX.Y.Z — YYYY-MM-DD` header; leave `## Unreleased` empty.
7. `git tag -a vX.Y.Z -m "..."` on the release commit.
8. Push commit + tag to `origin` =
   github.com/ActiveInferenceInstitute/institute_website.

## Rules

- Tags are the authoritative release markers — every CHANGELOG-versioned release
  must have a matching annotated tag.
- `## Unreleased` must be emptied into a versioned section whenever shipped output
  reaches `main`.
- Contributors must always commit the rebuilt HTML (see
  [CONTRIBUTING.md](CONTRIBUTING.md)).

## History note

`v2.7.0` (2026-06-29) cut the domain migration + 11-locale i18n launch as a
**minor** release (public URLs were preserved via redirects). Annotated tags
`v2.5.0` and `v2.6.0`, previously missing, were backfilled at that time. Tags are
now the authoritative release markers — keep every CHANGELOG version tagged.
