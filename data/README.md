# `data/` — build provenance & project feed

Two committed inputs consumed by the build.

| File | Purpose |
| --- | --- |
| `export-manifest.json` | Provenance manifest of the upstream public-data export: per-file `name`, `output_path`, `record_count`, `sha256`, and `target`. `src/build.mjs` reads it to compute the `source_fingerprint` written into `version.json`. It records *what was exported and from where*, enabling reproducible-build verification. |
| `projects.json` | The public projects feed (id, title, summary, links, …) loaded via `loadProjectsData()` in [`../src/data.mjs`](../src/data.mjs) and rendered by [`../src/pages/projects.mjs`](../src/pages/projects.mjs) into `/projects/` and per-project pages. |

## Rules

- Both files are **public**. `projects.json` is part of the gated projection —
  it must contain only public-safe project rows (see [`../GATING.md`](../GATING.md)).
- `export-manifest.json` is regenerated by the upstream export pipeline; the
  `sha256` values are how a verifier confirms committed content matches the
  declared export. Do not hand-edit hashes.
- Changing `projects.json` is a content change: rebuild (`node src/build.mjs`)
  and commit the regenerated HTML together with the data edit.
