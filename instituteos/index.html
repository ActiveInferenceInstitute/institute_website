InstituteOS keeps operational knowledge in the private docs and library trees, then exports only public-safe slices into this static website. This page makes the interface explicit: authored public copy, sanitized public tables, exported graphs, public project pages, and the checks that block private material before publication.

Source Ownership

Private InstituteOS docs hold operating guidance, implementation notes, and verification records. The library tree holds registries for projects, policies, processes, communications, narratives, strategies, and graphs. The public website does not mirror those sources directly; it receives only authored public pages and exporter-produced public slices.

Open public mapView public projects

Export Boundary

All structured private-to-public data must pass through the InstituteOS website export pipeline or the public Open Source Map sync. The gate rejects contact details, direct private-platform destinations, private operational fields, raw task detail, nonpublic stewardship records, working documents, demos, recordings, and internal UI captures.

Best Public Interfaces

The site exposes the best public interface for each audience: curated pages for visitors, the resource directory for search and filtering, the Open Source Map for structured public rows, project pages for public project context, feeds for communications, and version metadata for provenance.

Resource directoryOpen Source MapVersion metadata

Release Checks

A public refresh is complete only after the sync/export checks, static build, internal-link check, site contract, static-security check, design-system check, and browser inspection pass. Generated root HTML is rebuilt from source, not hand-edited.

Public export gate

The private-to-public boundary is explicit

Private InstituteOS sources are projected into public artifacts only after the exporter, privacy gate, website checks, and browser inspection agree that the output is public-safe.

01

Author privately

Docs and library registries remain the source of truth for operational and governance work.

02

Project publicly

Exporter modules choose only public fields, public summaries, graph nodes, and approved brand assets.

03

Gate payloads

PublicGate rejects direct private-platform destinations, contact details, private keys, raw tasks, and working artifacts.

04

Build and verify

The website rebuilds clean URLs, search data, feeds, sitemap files, and provenance before release.

Visitors seeking an institutional orientation. About the Institute

The Active Inference Institute is a volunteer-led 501(c)(3) nonprofit dedicated to improving the accessibility, rigor, and applicability of Active Inference. We welcome people of all backgrounds, time zones, and familiarity levels.

 Visitors seeking to understand how the Institute is organized and governed. Institute Structure

The Active Inference Institute is organized around a governing Board of Directors, executive Officers, a Scientific Advisory Board, and two operational units for education and research.

 Builders, researchers, maintainers, partners, and contributors looking for concrete work. Projects

Projects convert shared ideas into repositories, research outputs, learning infrastructure, media, events, and applied work.

 Learners and researchers looking for structured entry points. Learning and Research

Learning resources point newcomers and experienced participants toward living references, livestreams, Active Inference Insights, textbook groups, readings, code implementations, and domain-specific courses.

 Visitors mapping domains, partners, projects, and application areas. Ecosystem

The Active Inference ecosystem includes challenge areas, user segments, information architecture, organizations, projects, and domains of application.
